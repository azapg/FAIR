### FAIR Platform: Storage Abstraction & Artifact Derivatives Implementation Spec

**Objective:** Refactor the existing tightly-coupled artifact system into a decoupled architecture. We are introducing a generic `StorageProvider` for byte-level storage (supporting local and S3 protocols to prioritize self-hosting and data sovereignty) and an `ArtifactDerivative` database model to handle complex, multi-file artifacts (like PDF submissions + AI transcriptions).

---

#### Phase 1: The Core Storage Abstraction

**Target File:** Create `src/fair_platform/backend/storage/provider.py`
**Dependencies:** `boto3` (needs to be added to `pyproject.toml` or `uv.lock`), `fastapi`, `pydantic`.

**Tasks:**

1. Define a `StorageProvider` Protocol using `typing.Protocol`.
* Methods required: `put_object(key: str, data: BinaryIO, content_type: str) -> str`, `get_object(key: str) -> BinaryIO`, `delete_object(key: str) -> None`, `get_presigned_url(key: str, expires_in: int = 3600) -> AnyUrl`.


2. Implement `LocalStorageProvider`.
* Use the existing directory logic from `src/fair_platform/backend/data/storage.py` (specifically `uploads_dir`) for physical storage.
* `get_presigned_url` should return a local API route (e.g., `/api/v1/storage/local/{key}`).


3. Implement `S3StorageProvider`.
* Initialize a `boto3` client using environment variables (`S3_ENDPOINT_URL`, `S3_ACCESS_KEY`, `S3_SECRET_KEY`, `S3_BUCKET_NAME`).
* This ensures out-of-the-box compatibility with AWS S3, Cloudflare R2, or local MinIO instances.


4. Create a factory function `get_storage_provider(config) -> StorageProvider` that returns the correct implementation based on a platform setting (defaulting to local).

#### Phase 2: Database Schema & Migrations

**Target Files:**

* `src/fair_platform/backend/data/models/artifact.py`
* New Alembic migration in `src/fair_platform/backend/alembic/versions/`

**Tasks:**

1. **Update `Artifact` Model:**
* Remove `mime`, `storage_path`, and `storage_type` columns from the `Artifact` model.
* Add a relationship: `derivatives: Mapped[List["ArtifactDerivative"]] = relationship(back_populates="artifact", cascade="all, delete-orphan")`.


2. **Create `ArtifactDerivative` Model:**
* Columns: `id` (UUID, primary key), `artifact_id` (ForeignKey to `artifacts.id`), `derivative_type` (String, e.g., 'original', 'transcription', 'annotated'), `storage_uri` (String, e.g., 'local://...', 's3://...'), `mime_type` (String), `created_at`, `updated_at`.


3. **Generate Migration:**
* Write an Alembic data migration that safely moves existing physical file references. For every existing `Artifact`, create an `ArtifactDerivative` where `derivative_type="original"`, `mime_type=artifact.mime`, and `storage_uri="local://" + artifact.storage_path`.

#### Phase 3: Refactoring the Artifact Manager

**Target File:** `src/fair_platform/backend/services/artifact_manager.py`

**Tasks:**

1. **Update Initialization:**
* Replace the `storage_backend=None` parameter with `storage_provider: StorageProvider`. Inject the provider from the factory.


2. **Rewrite `create_artifact`:**
* Instead of writing directly to disk, call `self.storage_provider.put_object(key, file.file, file.content_type)`.
* The `key` should be structured for namespace isolation: `artifacts/{artifact_id}/original_{filename}`.
* Insert the `Artifact` row, and then immediately insert the `ArtifactDerivative` row linking the generated `storage_uri`.


3. **Implement `add_derivative`:**
* Create a new method: `add_derivative(artifact_id: UUID, file: UploadFile, derivative_type: str, user: User) -> ArtifactDerivative`.
* This will allow AI grading workflows or external plugins to attach markdowns, rubrics, or summaries to existing submissions.


4. **Update Deletion Logic (`delete_artifact`):**
* If `hard_delete=True`, query all related `ArtifactDerivative` records.
* Loop through them, parse the URI, and call `self.storage_provider.delete_object(key)`.
* Finally, delete the database records.



---

#### Phase 4: Serving Files & Security Gatekeeping

**Target File:** `src/fair_platform/backend/api/routers/artifacts.py`

**Tasks:**

1. **Presigned URL Endpoint:**
* Create a new `GET /artifacts/{artifact_id}/derivatives/{derivative_id}/download` endpoint.
* Use `ArtifactManager.can_view` to enforce authorization.
* If authorized, fetch the `storage_uri`, parse the key, and return `StorageProvider.get_presigned_url(key)`.
* *Agent Note:* Ensure the endpoint returns a `307 Temporary Redirect` to the presigned URL, or returns the URL as JSON depending on frontend requirements.


2. **Local Storage Proxy (if local is active):**
* Create a protected route `GET /api/v1/storage/local/{key:path}` that serves files from the `uploads_dir` using FastAPI's `FileResponse`. This fulfills the presigned URLs generated by `LocalStorageProvider`.

#### Phase 5: Expanding the Abstraction (Avatars & Logs)

**Target Files:**

* `src/fair_platform/backend/data/models/user.py`
* `src/fair_platform/backend/data/models/workflow_run.py`

**Tasks:**

1. **Avatars:**
* Add an `avatar_uri: Mapped[Optional[str]]` column to the `User` model.
* Update user profile update endpoints to accept an image upload, stream it via `StorageProvider.put_object(f"avatars/{user_id}.jpg", ...)`, and save the URI.


2. **Workflow Logs:**
* Add a `log_uri: Mapped[Optional[str]]` column to `WorkflowRun`.
* Modify the workflow execution engine. Instead of appending log strings to a database cell, write them to a temporary file. On completion, upload via `StorageProvider.put_object(f"logs/runs/{run_id}.log", ...)` and store the URI.
